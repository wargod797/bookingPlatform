Endpoint Summary

Area Method Path Purpose Expected Success
Partner POST /partners/theatres Onboard a theatre for a city. 201 Created
Catalog POST /movies Create a movie. 200 OK
Catalog GET /movies List movies with optional genre and language filters. 200 OK
Catalog GET /movies/{id} Fetch movie details by ID. 200 OK or 404
Partner POST /partners/shows Create a show for a movie and theatre. 201 Created
Partner POST /partners/shows/{showId}/seats Allocate seat inventory for a show. 201 Created
Browse GET /browse/shows Browse shows by movieId, city, and date. 200 OK
Booking POST /bookings Book seats for a selected show. 200 OK
Booking GET /bookings/{id} Fetch a booking by ID. 200 OK or 404

Booking Flow

1
Onboard Theatre Create the city + theatre partner entry used by shows.
2
Create Movie Add a movie title with genre and language.
3
Create Show Link movie, theatre, date, time, and price.
4
Allocate Seats Register seat inventory for the show before booking.
5
Browse + Book Discover the show, book seats, then fetch the booking.
Important: /browse/shows returns an empty array until the movie, theatre, show, and seat inventory are created. The booking write path also requires the requested seats to exist for that show.

Scenario A: Offer Location + Afternoon Show

This scenario matches the selected city and theatre configuration and uses an afternoon show. Both discounts apply.

  • City: Mumbai
  • Theatre: PVR Andheri
  • Show time: 13:00
  • Seat count: 3
  • Ticket price: 100.0
  • Expected total: 200.0
curl.exe -X POST "http://localhost:8080/partners/theatres" `
  -H "Content-Type: application/json" `
  -d '{"theatreName":"PVR Andheri","cityName":"Mumbai"}'

curl.exe -X POST "http://localhost:8080/movies" `
  -H "Content-Type: application/json" `
  -d '{"title":"Interstellar","genre":"Sci-Fi","language":"English"}'

curl.exe -X POST "http://localhost:8080/partners/shows" `
  -H "Content-Type: application/json" `
  -d '{"movieId":1,"theatreId":1,"showDate":"2026-04-04","showTime":"13:00","price":100.0}'

curl.exe -X POST "http://localhost:8080/partners/shows/1/seats" `
  -H "Content-Type: application/json" `
  -d '{"seatNumbers":["A1","A2","A3","A4","A5"]}'

curl.exe "http://localhost:8080/browse/shows?movieId=1&city=Mumbai&date=2026-04-04"

curl.exe -X POST "http://localhost:8080/bookings" `
  -H "Content-Type: application/json" `
  -d '{"showId":1,"seats":["A1","A2","A3"]}'

curl.exe "http://localhost:8080/bookings/1"
Pricing calculation: (3 x 100 - 50) x 0.8 = 200.0. The third-ticket half-price offer applies first, then the afternoon 20% discount applies to the subtotal.

Scenario B: Non-Offer Location + Afternoon Show

This scenario uses an afternoon show outside the configured third-ticket offer location, so only the afternoon discount applies.

  • City: Delhi
  • Theatre: Downtown Screens
  • Show time: 13:00
  • Seat count: 3
  • Ticket price: 100.0
  • Expected total: 240.0
curl.exe -X POST "http://localhost:8080/partners/theatres" `
  -H "Content-Type: application/json" `
  -d '{"theatreName":"Downtown Screens","cityName":"Delhi"}'

curl.exe -X POST "http://localhost:8080/partners/shows" `
  -H "Content-Type: application/json" `
  -d '{"movieId":1,"theatreId":2,"showDate":"2026-04-04","showTime":"13:00","price":100.0}'

curl.exe -X POST "http://localhost:8080/partners/shows/2/seats" `
  -H "Content-Type: application/json" `
  -d '{"seatNumbers":["B1","B2","B3","B4"]}'

curl.exe "http://localhost:8080/browse/shows?movieId=1&city=Delhi&date=2026-04-04"

curl.exe -X POST "http://localhost:8080/bookings" `
  -H "Content-Type: application/json" `
  -d '{"showId":2,"seats":["B1","B2","B3"]}'

curl.exe "http://localhost:8080/bookings/2"
Pricing calculation: 3 x 100 x 0.8 = 240.0. Only the afternoon discount applies because the theatre and city are outside the configured offer list.

Validation Notes

  • Booking fails if any seat number is missing, duplicated, blank, or already booked.
  • The booking response includes createdAt once the booking is saved.
  • GET /movies/{id} is the only endpoint currently backed by Ehcache.
  • The third-ticket offer configuration is driven by booking.pricing.third-ticket-offer-cities and booking.pricing.third-ticket-offer-theatres.
